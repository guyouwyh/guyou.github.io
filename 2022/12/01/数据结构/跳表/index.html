# 跳表的思想

跳表的思想实际上与二分的思想类似，先介绍一下单链表，对于单链表来实说，查找的时间复杂度为 O (n), 如果是数组的话，查找的时间复杂度可以优化为 O (logn), 因为数组可以直接通过下标进行访问，但是链表不行，那么试着思考能不能建立多条链表来实现 O (logn) 查找呢？这就是跳表的思想，使用多条链表，用空间换时间的思想，去实现 logn 时间复杂度的查找。
下图为跳表的数据结构图:

可以看到，跳表实际上就是多条链表，分了很多层，这里姑且把层数成为 level。最小的 level 叫做最底层，最底层其实就是一条单链表，越往上，规定链表元素的数量必须是下一条链表的一半，这样空间复杂度实际上就是 O (n), 可以看到每一个节点有多个指针，这个指针的数量取决于跳表的最大 level。

# 跳表的查找

对于跳表的查找，其实是从最上层，开始进行匹配，当匹配到一个大于当前要查找的元素的时候，就往下一层继续查找。最终一定会找到底层。

# 跳表中元素的添加

跳表中的元素的添加比查找复杂点，因为跳表有多层，当添加一个元素的时候，上面的层数中是否也要添加？如果每层都要添加的话，那么会十分消耗性能的。这里其实可以采用一个概率的办法，对于最底层肯定是要添加的，至于上一层，添加该元素的概率为 0.5, 以此类推，这样就不必对所有的上层都添加元素。

# 跳表的删除

跳表的删除和查找很类似，所有层中的要删除元素，删除即可，但是有一点要注意，如果删除操作后导致上层中的元素个数为 0 了。那么要将记录的最大层数减少。
