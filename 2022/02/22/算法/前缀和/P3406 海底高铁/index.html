# 思路分析

由于题目中要考虑每段路是要买票还是要买卡，因此肯定要算出性价比，所以要得出每段路经过的次数。由于数据量是 1e5, 如果使用暴力算出每段路的经过次数，时间复杂度最坏情况下是 O (n^2)。因此必须优化下算每段路的经过次数。可以使用前缀和的思路，在每段路的起点 + 1, 在终点 - 1, 使用前缀和可以直接计算出每段路的经过次数。具体看代码！
