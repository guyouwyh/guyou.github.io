# 买票需要的时间

# 题目链接

https://leetcode-cn.com/problems/time-needed-to-buy-tickets/

# 思路分析

这是周赛第一题，算是一个签到题，计算出位于位置 k 的人完成买票需要的时间，由于每次买完一次票都需要排队，每次排队买票的顺序都是一定的。只要 ticket[i] 不为 0, 就可以继续买票，每进行一轮买票，就将每个人的票数都减去 1, 循环结束条件为第 k 个人买完票时。即 ticket[k]==0 .

# 反转偶数长度组的节点

# 题目链接

https://leetcode-cn.com/contest/weekly-contest-267/problems/reverse-nodes-in-even-length-groups/

# 思路分析

感觉这道题应该算是四道题中最难的，必须熟练掌握链表反转才能动手。
由于判定链表是否反转是通过链表的个数决定的，相当于将链表分为 1,2,3...n 组，每组的节点个数为组号 (除了最后一组外)。只有节点为偶数的组链表才需要反转。
对于链表的反转，可以使用迭代和递归两种方式，这里我使用的是迭代。
迭代反转链表的话需要一个 pre 指针指向前一个节点，一个 current 指针指向当前节点。
由于只是将部分链表反转，因此，还需要一个指针指向前一组的最后一个节点。一个指针指向后一组的第一个节点。head 指针指向当前组的第一个节点。
例如:
链表 ==[5,2,6,3,9,1]==, 除了要反转 2 和 6 之外，需要 5 指向 6 , 2 指向 3 。

解题步骤:

  1. 先遍历 count 次链表 (count 取 1,2,3,4....), 得出链表的长度 c。此时 pre 应该指向当前组的最后一个节点，而 current 指向下一组的第一个节点。

  2. 根据 c 判断是否需要反转，即 c 为偶数。
    如果需要反转，由之前保存的 head 指针，可以从当前组的第一个节点开始进行反转。反转完之后，current 指针指向下一组的第一个节点，pre 指针指向当前组的最后一个节点。此时需要前一组的最后一个节点的指针 tail 和当前组的第一个节点实现与前后两组链表连接。 tail.next=pre;head.next=current 。完成之后要更新 tail 指针，此时 head 原来是当前组的一个节点，链表反转之后成为了最后一个节点。因此 tail 指针指向 head。
    如果不需要反转，只要更新 tail 的值即可，将 tail 指向 head, 然后右移 c-1 位。即指向了当前组的最后一个节点。

  3. 更新 head 指针，head 指针应该指向下一组的第一个节点。将 count+1, 并将链表长度 c 置为 0。

# 题目链接

https://leetcode-cn.com/contest/weekly-contest-267/problems/process-restricted-friend-requests/

# 思路分析

通过审题可以得出，这题是一道并查集模板题目，但是又需要进行一些条件判断。在 u 和 v 建立关系之前，这里将不能建立关系列表称为黑名单列表，需要判断他们两个建立好关系后会不会影响黑名单列表中的人建立关系。
如果 u 和 v 建立关系会导致黑名单列表中的人建立关系，说明他们两个不能建立关系，反之则可以建立关系。
