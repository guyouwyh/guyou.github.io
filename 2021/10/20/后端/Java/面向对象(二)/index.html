# static 关键字

statci 可以用来修饰属性、方法、代码块、内部类

static 修饰属性：静态变量

实例变量：当创建类的多个对象时，每个对象都独立的拥有一套类中的非静态属性，。

静态变量：当创建了类的多个对象时，多个对象共享同一静态变量，当某个对象修改静态变量时，其他对象的静态变量也会随之改变。

  • 静态变量随着类的加载而加载
  • 静态变量的加载要更早于对象的创建
  • 由于类只会加载一次，则静态变量在内存也只会加载一次，存在方法区的静态域中。
  • 静态变量可以不用创建对象，直接通过类名去调用

statci 修饰方法：静态方法

  • 随着类的加载而加载。
  • 通过类名可以直接调用

注意：在静态的方法内，不能使用 this 关键字、super 关键字，静态方法里只能调静态

# main 方法

  • main（）方法作为程序的入口
  • main（）方法也是一个普通的静态方法
  • main（）方法可以作为我们与控制台交互的方式

# 代码块

代码块的作用：

  • 初始化类或对象
  • 如果有修饰的话，只能使用 static

静态代码块和非静态代码块

静态代码块：

  • 内部可以有输出语句
  • 随着类的加载而执行，而且只执行一次
  • 如果一个类中定义了多个静态代码块，则按先后顺序执行

非静态代码块：

  • 内部可以有输出语句
  • 随着对象的创建而执行
  • 每创建一个对象非静态代码块就能执行一次

注意：当子父类中同时有构造代码块、静态代码块、构造方法时候，执行顺序为

1. 父类静态代码块

2. 子类静态代码块

3. 父类构造代码块

4. 父类构造方法

5 子类构造代码块

6. 父类构造方法

# final 关键字

final 可以用来修饰类、方法、变量

final 修饰类：不能被其他类继承

final 修饰方法：方法不能再被重写了

final 修饰属性：变量变为常量

​ 赋值位置：显式初始化、代码块中初始化、构造器中初始化

final 修饰局部变量：尤其是 final 修饰形参时，一旦赋值以后就只能在方法体内使用，而不能修改

static final 修饰属性属性：全局常量

# 抽象类

abstract 关键字，可以用来修饰类、方法

修饰类：抽象类

  • 此类不能实例化
  • 类中一定有构造器，便于子类实例化时调用

修饰方法：修饰方法

  • 方法没有方法体了

  • 包含抽象方法的类一定是抽象类，类是抽象类中不一定有抽象方法

  • 若子类重写了父类中的所有抽象方法后，子类才能实例化，否则子类也属于抽象类

注意：

  • abstract 不能用来修饰：属性、构造器等结构
  • abstract 不能修饰私有方法、静态方法、final 的方法（都不能重写方法）

# 接口

interface 关键字

接口的使用：

  • 接口使用 interface 定义

  • java 中，接口和类是两个并列的结构

  • 如何定义接口，定义接口中的成员

    • JD7 及以前，定义全局常量和抽象方法

      1. 全局常量：public static final（可以省略）

      2. 抽象方法：public abstract（可以省略）

    • JDK8：还可以定义静态方法和默认方法

  • 接口中不能定义构造器

  • 实现接口用 implements 来实现

  • 可以实现多个接口，弥补了继承性的局限性

  • 接口和接口之间也可以继承

# 抽象类和接口的异同：

一、表示内容不同

1、抽象类：抽象类表示该类中可能已经有一些方法的具体定义。

2、接口：接口就仅仅只能定义各个方法的界面（方法名，参数列表，返回类型），并不关心具体细节。

二、性质不同

1、抽象类：抽象类是对象的抽象。

2、接口：接口是一种行为规范。

三、成员变量不同

1、抽象类：抽象类中的成员变量可以被不同的修饰符来修饰。

2、接口：接口中的成员变量默认的都是静态常量（static final）。

# 内部类

java 中允许将一个类 A 声明在另一个类 B 中，则类 A 就是内部类，类 B 称为外部类

每个内部类都能独立继承一个类或实现一个接口，无论外围类是否已经继承某个 (接口的) 实现，对于内部类都没有影响。

接口只是解决了部分问题，而 内部类 使得多重继承的解决方案变得更加完整。

内部类的特性

  1. 内部类可以用多个实例，每个实例都有 自己 的状态信息，并且与其他外围对象的信息相互独立
  2. 在单个外围类中，可以让多个内部类以不同的方式实现同一个接口，或者继承一个类
  3. 创建内部类对象的时刻并不依赖于外围类对象的创建。
  4. 内部类提供了更好的封装，除了该外围类，其他类都不能访问

分类：

  • 匿名内部类
  • 成员内部类 (静态和非静态)
  • 局部内部类
  • 静态内部类

成员内部类

成员内部类是最普通的内部类，可以无限制的访问外围类的所有成员属性和方法，尽管是 private 的，但外部类访问内部类的成员属性需要通过内部类实例来访问。

注意:

  1. 成员内部类中不能存在任何 static 的变量和方法。
  2. 成员内部类是依附于外部类的，所有只有先创建了外围类才能够创建内部类。

局部内部类

局部内部类是嵌套在方法和作用域内的，它只能在该方法和属性中被使用，出了该方法和属性就会失效。

注意：在局部内部类的方法中，如果调用局部内部类所声明的方法中的局部变量，要求此局部变量声明为 final（可以从生命周期的角度理解）

因为 method 方法一旦执行完，局部变量 a 就会消失，从而影响内部类，因此 成员内部类中调用外部的变量必须是常量，即用 final 修饰。

  • 静态内部类不能使用外部类的非 static 方法
