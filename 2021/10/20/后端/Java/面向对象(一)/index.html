# 关键字

被 Java 语言赋予了特殊含义，用做专门用途的字符串

关键字的特点

  • 完全小写的字母
  • 在开发工具中有特殊颜色

用于定义数据类型的关键字:

class、interface、enum、byte、short、int、long、float、double、char、boolean、void

用于定义流程的关键字:

if、else、switch、case、default、while、do、for。break、continue、return

用于定义访问权限修饰符的关键字:

private、protected、public

# 保留字

现有 Java 版本尚未使用，但以后版本可能会作为关键字使用

goto、const

# 标识符

标识符是指在程序中，我们自己定义的内容

命名规则：

  • 标识符可以包含英文字符、数字、 $ 和 _
  • 标识符不能以数字开头
  • 标识符不能是关键字
  • 不能包含空格

命名规范:

  • 类名规范：首字母大写，后面每个单词首字母大写
  • 变量名规范：首字符小写，后面每个单词首字母大写
  • 方法名规范：同变量名

# 变量

变量的概念

  • 内存中的一个存储区域
  • 该区域的数据可以在同一类型范围内不断变化
  • 变量是程序中最基本的存储单元，包含变量类型、变量名和存储的值

变量的作用

用于在内存中保存数据

使用变量注意:

  • Java 中每个变量必须先声明、后使用
  • 使用变量名来访问这块区域的数据
  • 变量只有在其作用域内才有效
  • 同一作用域内，不能定义重名的变量

# 数据类型

数据类型分为基本数据类型和引用数据类型

# 基本数据类型

数值型:byte (1 字节)、shor (2 字节) t、int (4 字节)、long（8 字节）

浮点型:float (4 字节)、double（8 字节）

布尔型:boolean

# 引用数据类型

类:class

接口:interface

数组:[]

字符串：String

整型：

  • java 的整型常量默认为 int 型，声明 long 型常量需要加 l 或 L。
  • java 程序中变量通常声明为 int 型，除非不足以表示较大的数才用 long

浮点型：

  • java 的浮点型常量默认为 double 型，声明 float 型常量，必须加 f 后或 F
  • float 表示数值的范围比 long 还大

# 类型转换

当数据类型不一样时，将会发生数据类型转换

自动类型提升 (隐式)：当容量小的数据类型的变量与容量大的数据类型的变量做运算时，结果自动提升为容量大的数据类型

byte、char、short->int->long->float->double

  • 代码不需要经过特殊处理，自动完成
  • 规则：数据范围从小到大

强制类型转换 (显式)

  • 代码需要进行特殊的格式处理，不能自动完成
  • 范围小的类型 范围小的变量名 =（范围小的类型）原来范围大的数据

注意事项:

1. 强制类型转换一般不推荐使用，因为有可能发生精度损失、数据溢出。

2.byte/short/char 这三种类型都可以发生数学运算

3.byte/short/char 这三种类型在运算的时候，都会被首先提升成 int 类型，然后进行计算。

4.boolean 类型不能发生数据类型转换

& 和 && 的区别:

如果前面一个变量是 false, && 不会对后面的变量进行判断，而 & 会进行判断，但是运算的结果是相同的。

| 和 || 的区别:

如果前面一个变量是 true, || 不会对后面的变量进行判断，而 | 会进行判断，但是运算结果是相同的

# 面向对象

面向对象的三大特征：

  • 封装性
  • 继承性
  • 多态性

面向对象和面向过程的区别

面向过程：强调的是功能和行为，以函数为最小单位，考虑怎么做

面向对象：强调具备了功能的对象，以类 / 对象为最小单位，考虑谁来做

面向对象的两个要素：

  • 类：对一类事物的描述，是抽象的、概念上的定义
  • 对象：是实际存在的该类事物的每个个体，因而也成为实例

类的组成：

属性 / 成员变量（field）

方法 / 成员方法（method）

创建类的对象 = 类的实例化 = 实例化类

# java 的内存解析

  • 堆（Heap): 存放对象实例

  • 栈（stack): 指虚拟内存栈，存储局部变量

  • 方法区 (Mehtod Area): 用于存储已被虚拟机加载的类信息、常量、静态变量、即时编译器编译后的代码等数据

类中的属性和局部变量的区别：

不同点：

1. 在类中声明的位置的不同

属性：直接定义在类的 {} 内

​ 局部变量：声明在类的方法内、方法形参、代码块内、构造器形参、构造器内部的变 量。

2. 关于权限修饰符的不同

​ 属性：可以在声明属性时，指明其全新啊，使用权限修饰符，常用的权限修饰符有：private、public、protected

​ 局部变量：不可以使用权限修饰符

3. 默认初始化值情况

​ 属性：类的属性，根据其类型都有默认初始化值（引用数据类型的默认初始值都是 null）

​ 局部变量：没有默认初始化值。（形参在调用时赋值即可）

4. 内存中加载的位置不同

属性：加载在堆空间中（static 加载在方法区中）

局部变量：加载到栈空间中

相同点：

1. 定义变量的格式都是一样的

2. 先声明后使用

3. 变量都有其对应的作用域

# 对象数组的内存解析

匿名对象：new 出的一个对象没有引用，特点是只能一次性使用

# 方法重载

方法重载：在同一个类中，允许存在一个以上的同名方法，只要它们的参数个数或者参数类型不同即可。

口诀：两同一不同：同一个类，同一个方法名，参数列表不同。

判断是否重载：跟方法的权限修饰符、返回值类型、形参变量名、方法体都没有关系

# 值传递机制（方法的参数传递机制）

  • 如果参数是基本数据类型，此时实参赋给形参的是实参真是存储的数据值
  • 如果参数是引用数据类型，此时实参赋给形参的是实参存储数据的地址值

# 封装性

高内聚：类的内部数据操作细节自己完成，不允许外部干涉

低耦合：仅对外暴露少量的方法用于使用

权限修饰符：private、缺省（什么都不写）、protected、public

# 构造器（构造方法）

作用：

  • 创建对象
  • 初始化对象的属性

特点：1. 默认构造器为无参构造器

​ 2. 一个类中至少有一个构造器

​ 3. 一个类中定义的多个构造器，彼此构成重载。

定义构造器的格式：权限修饰符 类名 (形参列表){}

javabean 指符合如下标准的 java 类

  • 类是公共的
  • 有一个无参的公共构造器
  • 有属性，且有对应的 get、set 方法

# this 关键字的使用

this 可以修饰属性、方法、构造器

this 修饰属性和方法的时候可以理解为当前对象

在类的方法中，我们可以使用”this.“的方式来调用对象的方法和属性

this 调用构造器：

注意：

  • 构造器中不能通过 this（形参列表）方式调用自己

  • 如果一个类中有 n 个构造器，最多有 n-1 个构造器使用了 this（形参列表）

  • this 调用构造器必须是在首行

# package 关键字

  • 为了更好实验项目中的类的管理，提供包的概念
  • 使用 package 声明类或接口所属的包，声明在源文件的首行
  • 包，属于标识符，遵循标识符的命名规则、规范（xxxyyyzzz）
  • 每 '.' 一次代表一层文件目录

注意：同一个包下不能命名同名的接口、类，不同包下可以命名同名的接口、类

# import 关键字的使用

  • 在源文件中显式的使用 import 结构导入指定包下的类、接口

  • 声明在包的声明和类的声明之间

  • 如果需要导入多个结构，则并列写出即可

  • * 代表可以导入某包下的所有结构

  • 如果使用的类或接口是 java.lang 包或本包下定义的可以不用 import

  • 如果在源文件中使用了不同包下同名的类，则必须至少有个类用全类名的方式显示

  • * 只能表示内部的接口和类，不能使用子包

# 继承性

继承的好处：

  • 减少了代码的冗余，提高了代码的复用性
  • 便于功能分扩展
  • 为多态性的使用提供了前提

继承的格式：class A extends B {}

A：子类、派生类

B：父类、超类、基类

体现：一旦子类继承父类 B 后，子类 A 中就获取了父类 B 中声明的结构、属性、方法

注意：父类中声明的 private 的属性和方法，子类继承父类后，仍然认为获取了父类中私有的结构，只是因为封装性的影响，使得子类不能直接调用父类的结构而已。

java 中继承的规定：

  • 一个类可以被多个子类继承

  • 一个类只能有一个父类，单继承性

  • 类可以多层继承

  • 子类直接继承的父类称为直接父类，间接继承的父类称为间接父类

  • 所有类除 java.lang.Object 类都直接或间接继承与 java.lang.Object 类

# 方法的重写

定义：子类对父类方法的重置和覆盖。

要求：

  • 重写的方法必须与父类被重写的方法具有相同的方法名称、参数列表

  • 返回值类型不能大于父类被重写的返回值类型（子类重写返回值类型必须是被重写返回值类型的子类，被重写的类型的如果是基本数据类型，则重写的必须也是基本数据类型）

  • 子类方法抛出的异常不能大于父类被重写方法的异常

  • 只有非 static 关键字才能重写

# super 关键字

super 可以当做父类的对象。

super 可以用于在子类中调用父类的属性、方法、构造器。

  • 可以在子类方法或构造器中，通常使用 super. 属性或 super. 方法的方式，显式调用父类中声明的属性和方法
  • 当父类和子类定义了同名的属性时，我们要在子类中调用父类的属性必须用 super。

super 调用构造器

  • 可以用 super（形参列表）调用父类中声明的指定构造器
  • super（形参列表）必须声明在子类构造器的首行
  • 在类的构造器中，this（形参列表）和 super（形参列表）只能写一个
  • 在类的构造器中，如果不写 super 或 this 默认是 super
  • 在类的多个构造器中，至少有一个构造器中使用 super（形参列表）调用了父类构造器

# java 中子类对象的实例化过程

子类对象在实例化过程中，子类对象会默认先调用父类中的无参构造，然后调用子类中的中的构造方法

事实上：在子类构造方法的首行相对于默认隐含了一个 super（）语句。

虽然创建子类对象时，调用了父类的构造器，但是自始至终只创建过子类一个对象。

# 多态性

对象的多态性，父类的引用指向子类的对象

多态的使用：当调用子父类同名同参数的方法时，实际执行的是子类重写父类的方法。

在编译期只能调用父类中声明的方法，但运行期只能执行子类重写父类的方法。

口诀：编译看左，运行看右

多态性的使用前提：

  • 有类的继承关系
  • 方法的重写

对象的多态性只适用于方法，不适用于属性。

属性：编译和运行看左边

方法：编译看左，运行看右

多态性是运行时行为

# == 和 equals 的区别

==：运算符

  • 可以使用在基本数据类型变量和引用数据类型变量中

  • 如果比较的是基本数据类型变量，比较两个变量保存的数据是否相等（类型不一定要相同）

  • 如果比较的是引用数据类型变量，比较两个对象的地址值是否相同（即两个引用是否指向同一个对象）

equals：一个方法

  • 只能适用于引用数据类型
  • object 中 equals 与 == 作用相同
  • 如果重写了 equals，比较的不一定是地址相同

# tostring 方法的使用

  • 当输出一个对象的引用时，实际上就是调用当前对象的 tostring 方法

  • 像 String、Date、file、包装类等都重写了 object 中的 tostring 方法，在调用 tostring 时候返回实体内容。
