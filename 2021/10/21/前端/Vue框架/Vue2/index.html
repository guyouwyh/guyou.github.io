# vuejs 的概述

Vuejs 是一套构建用户界面的框架，它只关注视图层的内容，它是前端的主流框架之一。

# MVVM 思想

MVVM 是前端视图层的分层开发思想，主要是用来将没一个页面分层 M,V,MV 这三种组件。这三个组件在前端开发中分工协作，分工明确，其中 VM 是 MVVM 的核心，是因为 VM 是 V 和 M 之间的一个整体调度这，vue.JS 框架创建 VM 这一组件，来为 M 和 V 来进行服务

# v-once

该指令表示元素和组件只会渲染一次，不会随着数据的改变而改变

如果显示的信息后续不需要再修改，可以使用 v-once

# v-cloak

主要是为了解决差值表达式的闪烁问题

使用差值表达式的问题：在页面加载的过程中，在页面中的 {{}}`插值表达式首先会被页面认可为是html元素中的实际存在的内容。所以浏览器会首先将`{{}} 展现在页面上，页面加载完毕后，差值表达式的 {{}}`才会真正的转变为动态赋予的值。这就会造成一种现象，如果将来终端来访问服务器的过程中，网速较慢，那么我们的`{{}} 会首先展示出来，会一闪而过，最终显示的是被赋予的值。

v-cloak 属性在还没被 js 代码解析的时候，不会消失，这时候设置样式为不可见，当被 js 代码解析后就会消失，这是插值表达式已经被赋值了，样式也不起作用了。

# v-text

# V-html

插值表达式与 v-text/v-html 的区别

  • 对于元素中已经存在的值，只有差值表达式能够将原有的值保留，在原有的已经存在的值的基础上添加动态数据

  • 使用 v-text 和 v-html 之所以不能够保留元素标签中原有的值是因为在使用以上两个指定属性之前，会提前将标签对中的内容清空掉。

  • v-text 主要是用来赋予纯内容的，如果赋予的元素中的内容包含 html，那么 v-text 会将 html 原封不动的展示在页面上

  • v-html 除了能将元素赋予内容外，如果本身含有 html 代码，那么复制后最终展现出来的结果会被浏览器解析。

# v-pre

填充原始信息，显示原始信息，跳过编译过程

# v-bind

v-bind：是 vue.js 中提供用来绑定属性的指令

v-bind 使用频率很高，可以直接简写成 : 的方式

v-bind 也可以使用拼接字符串的形式，使用 +

可以控制 class 的属性

一般通过 vue 对象中的属性动态绑定值，如果为 true, 该属性就会添加，为 false 就不会添加。

# v-on

对事件的绑定

v-on:click="fun1"

可以使用 @ 符号来简写

@click="fun1"

参数传递问题

  • 事件调用的方法有参数时候，Vue 会默认将浏览器生产的 event 事件对象作为参数传入到方法。
  • 如果我们需要 event 对象又需要其他参数，可以使用 $event 获取 event 对象

# v-on 事件修饰符

# .stop

.stop 对时间的冒泡机制进行阻止，阻止扩散冒泡的行为

js 中的事件的冒泡指的是，在触发了内层元素的同时，也会随之触发外层元素 (外层元素包裹了内层元素)。

# .prevent

使用.prevent 阻止超链接的默认行为

所谓默认的行为指的是超链接的 href 属性。

# .capture

实现捕获触发事件的机制。在嵌套事件时，被.capture 修饰的事件会优先触发

# .self

阻止冒泡，阻止自身冒泡的行为

# .once

使用.once 修饰符，只触发一次事件处理函数。

.once 需要结合.prevent 来使用

当添加上 @click.prevent.once 之后，点击超链接

第一次点击：只触发了 @click 事件，href 链接没有触发

第二次点击：没有触发 @click, 只触发了 href

Vue 在进行 dom 渲染的时候，出于性能的考虑，会尽可能的复用已经存在的元素，而不是重新创建元素。我们可以添加 key 属性来解决重复利用问题。

# 双向数据绑定

单项数据绑定的方式，Model 去渲染 View.

v-bind 只能实现数据的单向绑定。

双向数据绑定的概念：

所谓的双向数据绑定是指：不仅仅只是从 Model 取得数据渲染到 View，还可以从 View 的变化动态更新 M 的值
