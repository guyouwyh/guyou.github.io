# SpringMvc 执行流程

DispatchServlet 前端控制器，在符合拦截路径中的所有请求都会经过这个 Servlet

进入源码

进入到 doDispatch 中

请求处理的流程总结

1. 所有请求首先被 DispatchServlet 进行接收

2.DispathServlet 调用 doService 方法

3. 在 doservice 中调用 getHandle (): 根据请求地址获取能处理该请求的处理器

4.getHandlerAdapter (): 根据当前的处理器来获取能调用处理器所有方法的处理器适配器]

5. 执行目标方法后会返回一个 ModelAndView 对象

6. 根据 ModelAndView 对象中的信息转发到相应的页面，并可以在请求域中取出 ModelAndView 中的模型数据

getHandle 具体细节：

HandlerMappingg ：处理器映射器，其中有一个子类是 DefalutAnnotationHandler , 其中有一个 HandleMap , 保存的请求路径和相对应的处理器信息， HandleMap 在 ioc 容器启动创建 Controller 对象的时候扫描每个处理器都能处理什么请求。保存在 HandlerMapping 的 handleMap 属性中。

getHandlerAdapter (): 具体细节

与 handleMapper 类似，循环遍历每一个 handlerAdapters, 如果找到支持处理器的，就返回.

# mvc:annotation-driven

mvc:annotation-driven 注解的作用

  • 自动注册 RequestMappingHandlerMapping （动态资源可以访问）

  • 自动注册 RequestMappingHandlerAdapter

  • 自动注册 ExceptionHandleExceptionResolver

  • 支持使用 ConversionService 实例对表单参数进行数据类型转换

  • 支持使用 @Valid 注解对 JavaBean 实例对 JSR 303 验证

  • 支持使用 @RequestBody 和 @ResponseBody 注解

  • 支持使用 @DateTimeFormat 日期格式化注解和 @NumberFormat 数字格式化注解

# 格式化

@DateTimeFormat 日期格式化

参数属性为 patterm

标注在 javabean 类的 Date 属性上。参数为需要的日期格式，例如 yyyy-MM-dd

由于原来的 ConversionServiceFactoryBean 创建的 Conversion 中没有格式化器存在的，因此需要替换为 FormatConversionServiceFactoryBean

# 数据校验

前端校验很不安全，例如禁用 js 等可以绕过前端校验。

JSR303 是 java 为 Bean 数据合法性校验提供的标准框架

SpirngMvc 可以用 JSR303 来做数据校验

JDBC: 规范是实现各个厂商的驱动包

JSR303: 规范是 Hibernate Validator (第三方校验)

空检查

@Null 验证对象是否为 null
@NotNull 验证对象是否不为 null, 无法查检长度为 0 的字符串
@NotBlank 检查约束字符串是不是 Null 还有被 Trim 的长度是否大于 0, 只对字符串，且会去掉前后空格.
@NotEmpty 检查约束元素是否为 NULL 或者是 EMPTY.

日期检查

@Past 验证 Date 和 Calendar 对象是否在当前时间之前，验证成立的话被注释的元素一定是一个过去的日期
@Future 验证 Date 和 Calendar 对象是否在当前时间之后 ，验证成立的话被注释的元素一定是一个将来的日期
@Pattern 验证 String 对象是否符合正则表达式的规则，被注释的元素符合制定的正则表达式，regexp: 正则表达式 flags: 指定 Pattern.Flag 的数组，表示正则表达式

长度检查

@Size (min=, max=) 验证对象（Array,Collection,Map,String）长度是否在给定的范围之内
@Length (min=, max=) Validates that the annotated string is between min and max included.

数值检查

建议使用在 Stirng,Integer 类型，不建议使用在 int 类型上，因为表单值为 “” 时无法转换为 int，但可以转换为 Stirng 为”“,Integer 为 null
@Min 验证 Number 和 String 对象是否大等于指定的值
@Max 验证 Number 和 String 对象是否小等于指定的值

@Range (min=, max=) 被指定的元素必须在合适的范围内
@Range (min=10000,max=50000,message=”range.bean.wage”)

邮箱检查

@Email 验证是否是邮件地址，如果为 null, 不进行验证，算通过验证。
