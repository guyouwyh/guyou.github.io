# 请求处理

请求处理相关注解：

  • @RequestParam

  • @RequestHeader : 获取请求头中的某个值

  • @CookieValue : 获取 cookie 中 id 的值

    用于完成 Http 请求参数与业务方法参数的映射.

​ 相关参数:

​ value : 将 HTTP 请求中名为 num 的参数赋给形参 id。

​ requried : 设置是否为必填项，true 表示必填，false 表示可以不填

​ defalutValue : 如果 HTTP 请求中没有该参数，默认值为 0.

MVC 的自动封装

如果我们的请求参数是一个 pojo

SpringMVC 可以为我们直接为我们封装，如果 pojo 中还有另一个 pojo2, 可以在页面中的 name 属性值填写 pojo2. 属性值，也可以为我们进行级联封装.

也可以直接写原生 API, 比如 HttpSession,HttpRequest，httpResponse

# 转发和重定向

转发 ：

处理器方法返回 ModelAndView 实现转发 forward

语法：setViewName ("forward: 视图文件完整路径")

forward 特点：不和视图解析器一同使用，就当项目中没有视图解析器

可以转发到视图解析器以外的界面

重定向 :

框架会把 model 中的简单数据类型，转为 string 使用，作为重定向页面的 get 请求参数使用。目的是在两次请求之间传递数据

可以使用参数 ${param.} 获取请求参数的值

不能访问 web-inf 下的资源

如果在控制器中返回 String 的话，如果使用 return "index" 的话，会经过视图解析，默认是转发，

如果使用 return "forward:" 需要写全路径，既可以转发到页面，也可以转发到其他控制器，

如果使用 return "redirect:" , 是不经过视图解析器，既可以转发到页面，也可以转发到其他控制器

# 异常处理

springmvc 框架采用的是统一，全局的异常处理。

SpringMVC 中三个默认的异常解析器

  • ExceptionHandlerExceptionResolver (使用注解驱动后才有): 处理 @ExceptionHandler 注解
  • ResponseStatusExceptionResolver ：处理 @ResponseStatus 注解
  • DefalutHandlerExceptionResolver : 判断是否 SpringMVC 自带异常

如果三个异常解析器都不能解析，就抛出异常给 tomcat

通过 HandlerExceptionResolver 处理程序的异常，包括 Handler 映射，数据绑定以及目标方法执行时的异常.

把 controller 中的所有异常处理都集中到一个地方，采用的是 aop 的思想，把业务逻辑和异常处理代码分开。解耦合。

使用两个注解

  • @ExceptionHandler , 告诉 springmvc 这个方法专门处理异常，参数是异常的 class，当发生此类型的异常时。

    • 由当前方法处理，如果没有参数，就是处理其他异常。
    • 处理异常的方法的参数中可以使用 Exception 来接收产生的异常，携带异常信息不能给参数位置写 model, 但是返回 ModelAndView
    • 精确处理异常

  • @ControllerAdvice ：写在类上，将集中处理所有异常的类加入 IOC 容器中

通过 ResponseStatusExceptionResolver 处理异常

** 使用注解 @ResponseStatus**

参数

  • reason ：自定义异常原因

  • value ：跳转异常页面

    • HttpStatus. 错误类型

异常处理发生的逻辑：

1. 需要把异常记录下来，记录到数据库，日志文件。

2. 发送通知，把异常的信息通过邮件、短信、微信发送给相关人员

3. 给用户友好提示

# 拦截器

拦截器是 springmvc 中的一中对象，需要实现 HandleInterception 接口

拦截器和过滤器类似，功能方向侧重点不同，。

过滤器是用来过滤请求参数，设置编码字符集等工作

拦截器是按揭用户的请求，对请求做判断

拦截器使用步骤：

1. 定义类实现 HandlerInterceptor 接口

2. 在 springmvc 配置文件中，声明拦截器

拦截器和过滤器的区别

1. 过滤器是 servlet 中规范中的对象，拦截器是框架中的对象

2. 过滤器是实现 Filter 接口的对象，拦截器是实现 HandleInterceptor 的

3. 过滤器是用来设置 request，response 的参数、属性，侧重对数据的过滤的拦截器是用来验证请求的

4. 过滤器是拦截器之前执行的

5. 过滤器是 tomcat 服务器创建的对象，拦截器是 springmvc 容器中创建的

6. 拦截器有三个执行时间点，过滤器有一个执行时间点

7. 过滤器可以处理 jsp、js、html 等拦截器是侧重拦截对 controller 对象，如果请求不被 DispatcherServlet 接收，拦截器不会拦截

单个拦截器的执行流程

1. preHandle 在方法调用前执行

2. postHandle 在方法调用后执行

3. afterCompletion 页面跳转后执行

多个拦截器执行流程

一个拦截器不放心，之后的流程都没有了。

按照 spring 中配置文件的顺序执行 preHandle , 如果前一个拦截器的 preHandle 方法放行，那么它的 afterCompletion 一定执行
