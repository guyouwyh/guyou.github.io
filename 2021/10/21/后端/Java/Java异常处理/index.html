# 异常分类

java 中，将程序执行中发生的不正常情况称为异常。

Error：java 虚拟机无法解决的严重问题，一般不便携针对性的代码进行处理

Exception：其他因编程错误或偶然的外在因素导致的一般性问题，可以使用针对性的代码进行处理

  • 空指针访问
  • 试图读取不存在的稳健
  • 网络连接中断
  • 数组越界

# 异常的处理过程

过程一；"抓"：程序在正常执行的过程中，一旦出现异常，就会在异常代码处生成一个对应异常类的对象，并将此对象抛出。一旦抛出对象以后，其后代码就不再执行。

过程二："抓"，可以理解为异常的处理方式：

①try-catch-finally

②throws

# try-catch-finally 的使用

注意：

  • finally 是可选的
  • 如果 catch 满足子父类关系，父类要先声明

异常对象处理方式：

①String getMessage

②printStackTrace()

体会：try-catch-finally 处理编译时异常，是待程序在编译时就不再报错，但是运行时仍可能报错。

开发中，由于运行时异常比较常见，我们通常不针对运行时异常进行 try-catch，一般处理编译时异常

# finally

  • finally 是可选的
  • finally 中声明的是一定会被执行的，即使 catch 中出现异常也会执行
  • 数据库连接、输入输出流、网络编程 Socket 等资源，jvm 是不能自动回收的，我们需要自己手动释放，需要声明在 finally 中
  • 先执行 finally 里的内容再执行 throws

# throws 的使用

throws + 异常对象

将异常对象抛出给方法的调用者，异常代码后续的代码就不再执行。

并没有真正的处理掉异常

# 方法重写异常

  • 子类重写的方法抛出异常类型不大于父类被重写的方法抛出的异常

  • 父类没有抛异常，子类也不能抛
