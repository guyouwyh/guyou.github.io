# StringTable

# String 的基本特性

  • String: 字符串，使用一对 "" 来声明
  • String 声明为 final 的，不可被继承
  • String 实现了 Serializable 接口，表示字符串是支持序列化的
  • String 实现了 Comparable 接口，表示 String 可以比较大小
  • String 在 jdk8 及以前内部定义了 final char [] value 用于存储字符串数据，jdk9 时改为 byte []
  • 通过字面量赋值是存储在常量池中

# 字符串常量池

<u> 字符串常量池中不会存储相同内容的字符串 </u>

  • String pool 是一个固定大小的 HashTable, 默认大小长度为 1009, 如果放进 String Pool 的 String 非常多，就会造成 Hash 冲突严重，从而导致链表很长。
  • 使用 - XX:StringTableSize 可设置 StringTable 的长度
  • 在 jdk6 中字符串常量池是固定的，是 1009 长度，设置没有要求
  • jdk7 中，字符串常量池默认是 60013, 设置没有要求
  • 在 jdk8 中，1009 是可设置的最小值

# String 中内存的分配

常量池类似于 Java 系统级别提供的缓存，8 种基本数据类型的常量池都是系统协调的，String 类型的常量池比较特殊，它的主要使用方法有两种

  • 直接使用双引号声明出来的 String 对象会直接存储在常量池中
  • 如果不是双引号声明的 String 对象，可以使用 String 提供的 intern () 方法存储到字符串常量池中

# 字符串拼接操作

  1. 常量与常量的拼接结果在常量池，原理是编译期优化

  2. 常量池不会存在相同内容的常量

  3. 只要其中一个是变量，结果就在堆中。变量拼接的原理是 StringBuilder

  4. 如果拼接的结果调用 intern () 方法，则主动将常量池中还没有字符串对象放入池中，并返回此对象地址

  5. 如果两个变量 (不包括用 final 修饰的常量引用) 拼接，实际上底层调用的是 StringBuilder, 将两个字符串 append, 然后通过 toString () 返回，在 jdk5.0 之后使用的是 StringBuilder, 在 jdk5.0 之前使用的是 StringBuffer

# 思考

String str=new String ("a")+new String ("b") 中间创建了几个对象？

对象 1:new StringBuilder ()

对象 2:new String ("a")

对象 3: 常量池中的 a

对象 4:new String ("b")

对象 5: 常量池中的 b

对象 6:toString () 中 new String ("ab")

# 关于 intern () 使用的总结

  • 在 jdk1.6 中，将这个字符串对象尝试放入常量池
    • 如果字符串常量池中有，则并不会放入，返回已有的池中的对象地址
    • 如果没有，会把此对象复制一份，放入池中，返回该地址
  • 在 jdk1.7 以后，将这个字符串对象尝试放入常量池
    • 如果字符串常量池中有，，则并不会放入，返回已有的池中的对象地址
    • 若有没有， 则会把对象的引用地址复制一份，放入池中，并返回池中的引用地址

如果需要多次使用重复的字符串，使用 new String (str).intern () 可以节省内存空间

# 概念

Direct Memory

  • 常见于 NIO 操作时，用于数据缓冲区
  • 分配回收成本较高，但读写性能高
  • 不受 JVM 内存回收管理

# 分配和回收的原理

  • 使用了 Unsafe 对象完成直接内存的分配和释放，并且回收需要主动调用 freeMemory 方法
  • ByteBuffer 的实现类内部，使用了 Cleaner (虚引用) 来监测 ByteBuffer 对象，一旦 ByteBuffer 对象被垃圾回收，那么就会由 RefenrenceHandle 线程通过 Clean 方法调用 freeMemory 来释放直接内存。
