# 思路分析

根据题意，需要我们判断 s 字符串中是否可以拆分成字典中出现的单词，字典中的单词可重复使用。所以我们需要从头到尾去遍历该字符串，判断字符串中是否字典中的单词。

# 方法一，字典树 + 记忆化搜索

对于单词的查找来说，可以使用一个数据结构 - 字典树。
下面介绍一下字典树:
字典树，又称为 Trie 树，也可以叫做前缀树，是一种树形结构，主要是用来统计和排序大量的字符串。
字典树的核心思想就是空间换时间，利用字符串的公共前缀来降低查询时间的开销以达到提高效率的目的。

字典树的三个基本的性质

  • 根节点不包括字符，除根节点外每一个节点都只包括一个字符.
  • 从根节点到某一节点，路径上经过的字符连接起来，为该节点对应的字符串
  • 每个节点的所有子节点包括的字符都不相同

假设 wordDict = ["cats", "dog", "sand", "and", "cat"]。那么构建出来的字典树如下图所示:

我们先构建好一个字典树，然后使用 dfs 去搜字典树，假设前 i 个字符组成的单词是在字典树中的，那么我们只需要判断 i 之后的字符是否处于字典树中，这时候就去搜索剩下的字符。但是注意，直接暴力搜索的很可能会超时，所以可以考虑用记忆化搜索去记录以某个字符开头的不可能在字典树种的情况。可以优化搜索时间。

# 方法二，动态规划

假设 dp [i] 代表 s [0~i-1] 组成的字符串是否在字典中，那么我们可以假设 j 为一个分隔点，如果 dp [j] 在字典中，那么我们只需要判断 s [j,i] 是否在字典中，如果在，那么 dp [i] 也一定在字典中。可以发现，dp [i] 依赖于 dp [j] 和 s [j,i]; 因此可以得到状态转移方程:
dp[i]=d[j]&&s.substring(i,j) 。
注意，我们需要初始化 dp [0] 为 true。
因为对于一个字符来说，它也可能是字典中的一个单词。所以 dp [1] 也依赖于 dp [0] 和 s [0,1];

字典树 + 记忆化搜索
